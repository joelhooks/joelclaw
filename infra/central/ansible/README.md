# Flagg Central Ansible scaffold

Tiny host-IaC layer for Flagg Central. This intentionally wraps the existing repo-managed scripts and launchd templates instead of replacing them with a giant config-management swamp.

## Why Ansible here

- macOS launchd is the real service manager.
- `community.general.launchd` can manage service enabled/running state.
- The existing `infra/central/scripts/*` already encode Central-specific checks, recovery, NAS proof, and smoke tests.
- Salt is too much fleet machinery for this shape right now.

## Install tools

Ansible is not currently installed on Flagg. Bootstrap from Joel's account:

```bash
brew install ansible
ansible-galaxy collection install -r infra/central/ansible/requirements.yml
```

## Dry run-ish validation

```bash
ansible-playbook \
  -i infra/central/ansible/inventory/flagg-local.ini \
  infra/central/ansible/playbooks/central.yml \
  --ask-become-pass \
  --check
```

`--check` is useful for file/template drift. Launchd install/disable/bootstrap is guarded behind `central_install_launchdaemons=true` so running the playbook does not accidentally stop an already-running Central stack.

For real launchd install without bootstrap:

```bash
ansible-playbook \
  -i infra/central/ansible/inventory/flagg-local.ini \
  infra/central/ansible/playbooks/central.yml \
  --ask-become-pass \
  -e central_install_launchdaemons=true \
  -e central_bootstrap=false
```

When Gate 4 bootstrap is approved:

```bash
ansible-playbook \
  -i infra/central/ansible/inventory/flagg-local.ini \
  infra/central/ansible/playbooks/central.yml \
  --ask-become-pass \
  -e central_install_launchdaemons=true \
  -e central_bootstrap=true
```

## Recovery helper

For the current blackhole class where ports accept but services don't answer:

```bash
sudo /Users/Shared/joelclaw/src/joelclaw/infra/central/scripts/recover.sh --all --passes 3
```

Or through Ansible, explicitly:

```bash
ansible-playbook \
  -i infra/central/ansible/inventory/flagg-local.ini \
  infra/central/ansible/playbooks/central.yml \
  --ask-become-pass \
  -e central_recover_now=true \
  -e central_recover_passes=3
```

The helper writes a timestamped receipt under `/Users/Shared/joelclaw/logs/central/` and exits non-zero unless all service probes pass consecutively.

## Scope

This scaffold owns boring host drift:

- service root directories and permissions
- launchd plist install/validation
- launchd enabled/running state
- preflight/health/reboot-proof invocation hooks

It does **not** own secrets. `.env` stays generated by `write-shadow-env.sh` / agent-secrets flow and remains uncommitted.
