Most personal AI projects start with a database. PostgreSQL, SQLite, maybe a vector store. You model your data, build an API, wire up auth.

I started there too. Then I asked a different question: _what if I own my own data server?_

## The substrate, not a feature

AT Protocol wasn't originally designed for agent systems. It was built for social networking â€” Bluesky runs on it. But the primitives are exactly what a personal AI system needs:

- **PDS (Personal Data Server)** â€” your own sovereign data store
- **DID identity** â€” cryptographic, portable, no passwords
- **Lexicons** â€” typed schemas for every record type
- **Firehose** â€” real-time stream of all changes
- **Federation** â€” servers talk to each other by design

Here's the thing that clicked for me: **the PDS _is_ the database**. You don't build an API in front of it. You don't bolt on auth. It already has identity, schema enforcement, and a real-time event stream baked in. Everything else just indexes or acts on PDS data.

## Everything gets a record

Is it overkill for one person's AI system? Maybe. But I've been burned enough times by "just throw it in a database and figure out the schema later." This way every piece of data has a type, a home, and a history from day one.

## Two halves of truth

The PDS holds agent data â€” structured, typed, federated. But there's a second half: the Obsidian Vault holds human knowledge â€” prose, wikilinks, architecture decisions, project notes.

Neither replaces the other. Qdrant indexes both. The agent reads from both. But they serve fundamentally different purposes:

- **PDS**: machine-written, machine-read, structured, versioned
- **Vault**: human-written, human-read, narrative, browseable

I need both. The agent needs both. Trying to shove everything into one system is how you end up with a junk drawer that nothing can actually reason over.

## Why federation matters for families

This is where it gets really interesting to me.

The endgame isn't one person with one agent. It's a family network:

- My PDS â€” the primary agent system

- Kids' PDSs â€” simplified agents, parental controls via Lexicon permissions

Each person owns their data. **Each person's agent can be as simple or complex as they want.** And because it's AT Protocol, anyone can move their PDS to different hardware. The identity follows. No vendor lock-in, no "one asshole" who can take it all away.

I'm repurposing a social media protocol as a personal operating system data layer. That's a bet. The ecosystem is immature for this use case. The Swift AT Protocol client libraries are thin. Running multiple PDS instances on one Mac Mini is going to be a pain in the ass to manage.
